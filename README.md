# TO-DO APP
Это веб-сервис на **Go** для управления списком дел с автоматическим расчетом дат. С

**Основные возможности:**
* **Управление задачами:** Создание, редактирование и удаление дел (CRUD).
* **Умный календарь:** Авторасчет следующей даты для повторяющихся задач.
* **Безопасность:** Защита API с помощью пароля и **JWT-токенов**.
* **Поиск:** Фильтрация задач по названию или описанию.
* **Docker** Полная готовность к развертыванию в изолированном контейнере.

# Запуск и компиляция Docker-образа

**1. Сборка бинарника (Linux x64):**
```bash
export CGO_ENABLED=0 GOOS=linux GOARCH=amd64
go build -o todo-app ./cmd/main.go
```

**2. Сборка образа:**
```bash
docker build -t todo-app .
```

**3. Запуск контейнера:**
```bash
docker run -d \
  --name todo-container \
  -p 7540:7540 \
  -e TODO_PORT=7540 \
  -e WEB_DIR=web \
  -e TODO_DBFILE=scheduler.db \
  -e TODO_PASSWORD=MyPassword123 \
  -e TODO_SECRET=MySecret123 \
  --mount type=bind,source="$(pwd)/scheduler.db",target=/app/scheduler.db \
  todo-app
```

# Архитектура проекта

Код организован по принципу разделения ответственности, и чистой архитектуры.

### Основные директории:
* **`cmd/`**: Точка входа в приложение. Здесь происходит инициализация зависимостей и запуск HTTP-сервера. Используется роутер `gorila/mux`.
* **`internal/`**: Основной код приложения, закрытый для импорта извне.
    * **`core/`**: Общая инфраструктура (настройка HTTP-сервера, общие доменные модели).
    * **`features/`**: Разделение по доменным областям (фичам): `auth` (авторизация) и `tasks` (задачи).
* **`web/`**: Статические файлы фронтенда (HTML, CSS, JS).

### Слои внутри каждой фичи (`features/`):
Каждая фича следует четкому пути прохождения данных:
1.  **Transport (HTTP)**: Обработка запросов, валидация входных данных (DTO), отправка ответов.
2.  **Service**: Ядро приложения. Здесь сосредоточена вся бизнес-логика.
3.  **Repository**: Слой доступа к данным. Отвечает только за выполнение SQL-запросов к SQLite.
